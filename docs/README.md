# Lazy A Build Manual

## North Star

Build a physical place inside a browser.

The visitor is borrowing someone's eyes.

## Core Laws

1. The room is real.
2. The room never performs.
3. The camera moves like a person.
4. The room is larger than the browser.
5. Only the image questions reality.
6. Reward attention, not clicks.
7. Never explain the trick.
8. Every interaction should feel inevitable, focused, and unexpected.

## Technical Philosophy

- HTML owns navigation and content.
- Three.js owns the room.
- Motion is authored, never game-like.
- Every camera move should complete in under ~800ms (except intentional hero moments).

## Folder Workflow

Each interaction folder contains:
- GOAL.md
- SPEC.md
- ACCEPTANCE.md

Do not begin the next interaction until the current one feels right.
